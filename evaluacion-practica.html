Evaluación Práctica: CRUD con Flet y Firebase

Propósito: Evaluar la capacidad del alumno para integrar una interfaz gráfica en Flet con servicios de Firebase Firestore mediante la REST API y autenticación anónima.

Esta evaluación requiere la creación de un sistema funcional que permita la gestión de datos (Guardar, Mostrar y Eliminar) asegurando la persistencia en la nube.

1. Competencias a evaluar

  • Implementación de Autenticación Anónima en Firebase.
  • Manejo de peticiones HTTP para interactuar con Firestore REST API.
  • Diseño de interfaces dinámicas y reactivas con Flet.
  • Gestión del ciclo de vida de los datos (CRUD básico).

2. Requerimientos del proyecto

  • Entorno: Python 3.10 o superior.
  • Librerías: flet y requests.
  • Firebase: Proyecto activo con Firestore y Auth habilitado.
  • REST API: Uso obligatorio de los endpoints de Firebase para operaciones.

3. Instrucciones de desarrollo

  1. Configurar en Firebase Console la Autenticación Anónima.
  2. Crear una base de datos Firestore en modo de prueba.
  3. Obtener la Web API Key desde la configuración del proyecto.
  4. Desarrollar la aplicación en Python utilizando el código base proporcionado.
  5. Completar las funciones de Guardar, Mostrar y Eliminar.

6. Rúbrica de Evaluación

Criterio Excelente (100%) Suficiente (70%)
Autenticación El login anónimo funciona correctamente y recupera el token. Implementado pero con errores de manejo de errores.
Operaciones CRUD Realiza satisfactoriamente Guardar, Consultar y Eliminar. Falta una de las tres funciones principales.
REST API Usa correctamente los métodos POST, GET y DELETE con requests. Uso incorrecto de headers o parámetros de URL.
Interfaz Flet Diseño limpio, usa variables de control y actualiza la UI. La UI no se actualiza tras realizar cambios.

7. Notas Técnicas Obligatorias

Es indispensable cumplir con lo siguiente antes de la revisión:

  • Verificar versión de Flet instalada: pip show flet.
  • Asegurarse de usar ft.run(main) al final de tu script.
  • Si trabajas en red escolar, verifica que el puerto 8080 esté disponible o usa el modo web.
