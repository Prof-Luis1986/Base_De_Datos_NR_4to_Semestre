🟦 Práctica 5

Consultas avanzadas y filtros en MongoDB (find, operadores y ordenamiento)

🧠 Guía de trabajo

En esta práctica aprenderás a consultar información específica dentro de una base de datos NoSQL usando filtros y operadores. Consultar bien los datos es tan importante como almacenarlos correctamente.

🧭 Contexto conceptual

Las consultas avanzadas permiten transformar datos en decisiones. Aprender a filtrar, ordenar y limitar es esencial para cualquier aplicación real.

🎯 Aprendizajes esperados

  • Filtrar documentos con condiciones.
  • Usar operadores como $gt, $lt.
  • Ordenar resultados.
  • Limitar resultados.
  • Analizar resultados de consultas.

🪑 Actividades en salón

Analiza:

{ "edad": { "$gt": 30 } }

Responde:

  • ¿Qué documentos regresaría?
  • ¿Para qué sirve este filtro?

💻 Actividades en laboratorio

(MongoDB – consultas)

Parte A – Filtros simples

db.usuarios.find({ edad: 30 }).pretty();

Parte B – Operadores de comparación

db.usuarios.find({ edad: { $gt: 30 } }).pretty();

db.usuarios.find({ edad: { $lt: 30 } }).pretty();

Parte C – Ordenamiento y límite

db.usuarios.find().sort({ edad: 1 }).pretty();

db.usuarios.find().limit(2).pretty();

🧪 Practicario

  • Realiza al menos 2 consultas con filtros.
  • Realiza 1 consulta ordenada.
  • Realiza 1 consulta con límite.
  • Explica qué devuelve cada una.

✅ Cierre

Al finalizar esta práctica sabes extraer información útil de una base NoSQL.

🧩 Justificación técnica de los ejercicios

Los ejercicios introducen filtros por rango y ordenamiento, habilidades básicas para construir búsquedas útiles en sistemas reales.

🌍 Escenarios reales de aplicación

  • E-commerce: catálogo, carrito y historial de compras.
  • Streaming: perfiles, reproducciones y recomendaciones.
  • Redes sociales: publicaciones, comentarios y reacciones.
  • Sistema escolar: alumnos, materias y evaluaciones.
  • Videojuegos: progreso, logros y sesiones de juego.

🔁 Comparación breve con modelo relacional

En SQL usarías WHERE, ORDER BY y LIMIT; en MongoDB esos conceptos se expresan con filtros JSON y métodos como sort y limit.

⚙️ Rendimiento y escalabilidad

  • Evita escanear colecciones completas cuando un índice puede ayudar.
  • Limita y proyecta campos para reducir transferencia de datos.
  • Cuida el tamaño del documento y el crecimiento de arreglos.
  • Mide consultas críticas y ajusta filtros antes de escalar.
  • Usa paginación o límites en lecturas de alto volumen.

🧪 Qué ocurre internamente en MongoDB

MongoDB genera un plan de consulta, decide si usar índices y ordena en memoria o usando el índice según el caso.

❗ Errores Comunes

  • Actualizar o eliminar sin filtro específico (o sin _id).
  • Guardar tipos incorrectos (números como texto, fechas sin formato).
  • Diseñar documentos demasiado grandes o con campos duplicados.
  • No documentar la estructura JSON y sus cambios.
  • Confundir colección con documento o arreglo con objeto.

✅ Buenas Prácticas Profesionales

  • Definir convenciones de nombres y mantenerlas en todo el proyecto.
  • Validar datos antes de guardar (tipo, rango, formato).
  • Usar índices en campos consultados con frecuencia.
  • Probar consultas con datos reales y revisar planes de ejecución.
  • Documentar decisiones de modelado y razones de embebido vs referencia.

🧠 Análisis Técnico y Reflexión

  • ¿Qué diferencia hay entre filtrar y ordenar?
  • ¿Cómo afecta el rendimiento ordenar sin índice?
  • ¿Qué impacto tiene limitar resultados en la UX?
  • ¿Qué filtro sería más selectivo y por qué?

🧭 Retos Aplicados

  • Combina dos filtros en una sola consulta y justifica su lógica.
  • Mide cuántos documentos devuelve cada consulta y explica por qué.
  • Aplica sort y limit juntos para simular un ranking.

🚀 Desafío Nivel Experto

  • Diseña un buscador con filtros múltiples y justifica cada condición.
  • Propón índices para acelerar las consultas más usadas.
