🟦 Práctica 2

Captura de datos y creación de documentos JSON usando comandos en MongoDB

🧠 Guía de trabajo

En esta práctica aprenderás a capturar información y transformarla en documentos JSON, utilizando comandos de MongoDB ejecutados desde MongoDB Studio 3T.

El objetivo es que comprendas cómo se crea una base de datos no relacional desde cero, cómo se insertan documentos JSON mediante comandos y cómo estos documentos forman parte de una colección. Antes de usar herramientas visuales o servicios en la nube, es indispensable entender qué hace cada comando y qué efecto tiene sobre los datos.

🧭 Contexto conceptual

Insertar datos es el primer punto de verdad del sistema: si el JSON está mal diseñado, todo lo demás falla. Esta práctica conecta captura de datos con persistencia real en MongoDB.

🎯 Aprendizajes esperados

Al finalizar la práctica, serás capaz de:

  • Crear una base de datos y una colección mediante comandos.
  • Insertar múltiples documentos JSON usando insertMany().
  • Consultar documentos con find().
  • Comprender cómo MongoDB almacena documentos JSON.
  • Relacionar comandos con cambios reales en la base de datos.

🪑 Actividades en salón

(Captura y análisis – sin computadora)

1. Captura de información

Se plantea el siguiente contexto: registro básico de usuarios de un sistema.

En tu cuaderno, captura la información de al menos 5 personas:

  • Nombre
  • Apellido
  • Edad

No pienses aún en MongoDB ni en comandos. Solo recopila los datos de forma clara.

2. Conversión de datos a estructura JSON

Con los datos capturados, responde:

  • ¿Qué datos se repiten?
  • ¿Qué campos serán las claves?
  • ¿Qué tipo de dato tiene cada valor?

Con base en esto, escribe un JSON base en tu cuaderno.

Ejemplo de referencia:

{
  "nombre": "",
  "apellido": "",
  "edad": 0
}
📌 Producto del salón: Datos capturados + JSON base definido.

💻 Actividades en laboratorio

(MongoDB Studio 3T – comandos)

Parte A – Creación de base y colección

Abre Studio 3T y crea una nueva ventana de consulta. Ejecuta los siguientes comandos:

use registro_nombres;

// Crea la colección
 db.createCollection("usuarios");

Verifica visualmente que la base y la colección existan.

Parte B – Inserción de documentos JSON

Inserta los documentos usando insertMany():

db.usuarios.insertMany([
  { nombre: "Juan", apellido: "Pérez", edad: 30 },
  { nombre: "María", apellido: "García", edad: 25 },
  { nombre: "Carlos", apellido: "López", edad: 40 }
]);

Consulta los documentos:

db.usuarios.find().pretty();

🧪 Practicario

Parte A – Comprensión de comandos

Explica con tus palabras qué hace cada comando:

  • use
  • createCollection
  • insertMany
  • find().pretty()

Parte B – Inserción ampliada

  • Inserta al menos 5 documentos adicionales, usando los datos capturados en el salón.
  • Ejecuta nuevamente: db.usuarios.find().pretty();

Responde:

  • ¿Cuántos documentos hay ahora?
  • ¿Todos tienen la misma estructura?

Parte C – Análisis de estructura

Identifica:

  • ¿Qué campos se repiten?
  • ¿Qué campo identifica de forma única a cada documento?

Explica por qué MongoDB genera automáticamente el campo _id.

🧩 Justificación técnica de los ejercicios

Los ejercicios de insertMany y creación de colecciones te obligan a respetar tipos, estructura y consistencia. Esto simula escenarios reales de carga inicial de datos.

🌍 Escenarios reales de aplicación

  • E-commerce: catálogo, carrito y historial de compras.
  • Streaming: perfiles, reproducciones y recomendaciones.
  • Redes sociales: publicaciones, comentarios y reacciones.
  • Sistema escolar: alumnos, materias y evaluaciones.
  • Videojuegos: progreso, logros y sesiones de juego.

🔁 Comparación breve con modelo relacional

En SQL, insertar implica respetar columnas y tipos fijos; en MongoDB puedes variar la estructura, pero debes controlar la calidad de los documentos.

⚙️ Rendimiento y escalabilidad

  • Evita escanear colecciones completas cuando un índice puede ayudar.
  • Limita y proyecta campos para reducir transferencia de datos.
  • Cuida el tamaño del documento y el crecimiento de arreglos.
  • Mide consultas críticas y ajusta filtros antes de escalar.
  • Usa paginación o límites en lecturas de alto volumen.

🧪 Qué ocurre internamente en MongoDB

En inserciones, MongoDB serializa a BSON, asigna _id, escribe en memoria y en el journal, y actualiza índices existentes.

❗ Errores Comunes

  • Actualizar o eliminar sin filtro específico (o sin _id).
  • Guardar tipos incorrectos (números como texto, fechas sin formato).
  • Diseñar documentos demasiado grandes o con campos duplicados.
  • No documentar la estructura JSON y sus cambios.
  • Confundir colección con documento o arreglo con objeto.

✅ Buenas Prácticas Profesionales

  • Definir convenciones de nombres y mantenerlas en todo el proyecto.
  • Validar datos antes de guardar (tipo, rango, formato).
  • Usar índices en campos consultados con frecuencia.
  • Probar consultas con datos reales y revisar planes de ejecución.
  • Documentar decisiones de modelado y razones de embebido vs referencia.

🧠 Análisis Técnico y Reflexión

  • ¿Qué campo usarías como identificador lógico además de _id?
  • ¿Qué problema surge si insertas estructuras distintas sin control?
  • ¿Cómo decidirías si un dato debe ser arreglo u objeto?
  • ¿Qué impacto tiene insertMany en rendimiento?

🧭 Retos Aplicados

  • Inserta documentos con campos opcionales y justifica cuándo omitirlos.
  • Agrega un subdocumento (direccion) y un arreglo (telefonos).
  • Crea un documento con fecha ISO y explica su tipo de dato.

🚀 Desafío Nivel Experto

  • Carga 20 documentos usando un patrón real (usuarios con preferencias) y documenta tu esquema.
  • Simula un error de tipo y explica cómo lo detectarías.
