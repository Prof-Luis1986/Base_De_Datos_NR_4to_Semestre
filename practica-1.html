📘 Práctica 1

Conexión profesional a MongoDB Atlas + Creación progresiva de base de datos

1. Propósito de la práctica

En esta práctica el alumno:

  • Configurará un entorno profesional en la nube usando MongoDB Atlas.
  • Conectará de forma segura con Studio 3T.
  • Comprenderá el flujo real de seguridad (usuario + IP + SSL).
  • Creará su propia base de datos.
  • Construirá colecciones y documentos elemento por elemento usando insertOne().

Esta práctica continúa directamente de la Práctica 1 (Relacional vs No Relacional) y marca el inicio del trabajo real en entorno profesional.

2. Aprendizajes esperados

Al finalizar, el alumno será capaz de:

  • Explicar qué es un cluster en la nube.
  • Configurar acceso seguro a una base de datos.
  • Usar una connection string SRV.
  • Conectarse mediante SSL.
  • Crear base de datos propia.
  • Insertar documentos JSON progresivamente.
  • Analizar la flexibilidad estructural de MongoDB.

PARTE I — CONFIGURACIÓN EN LA NUBE

3. Creación del Cluster (MongoDB Atlas)

  • Ingresar a MongoDB Atlas.
  • Crear proyecto (si no existe).
  • Database → Clusters → Create.
  • Seleccionar Free Tier (M0).
  • Elegir región cercana.
  • Esperar que el cluster esté activo.
📸 Evidencia: captura del cluster activo.

4. Configuración de Seguridad

A) Database Access

  • Security → Database Access.
  • Add New Database User.
  • Crear usuario: Equipo01 (o similar).
  • Definir contraseña por equipo.
  • Rol: Read and write to any database.
📸 Evidencia: usuario creado con rol asignado.

B) Network Access

  • Security → Network Access.
  • Add IP Address.
  • Permitir 0.0.0.0/0 para pruebas.
📸 Evidencia: regla de red configurada.

5. Obtención de Connection String

  • Clusters → Connect.
  • Drivers.
  • Copiar cadena SRV.
mongodb+srv://usuario:<password>@cluster.mongodb.net/?retryWrites=true&w=majority
⚠ Reemplazar <password> por la contraseña real.
📸 Evidencia: captura del apartado Drivers (sin mostrar contraseña).

PARTE II — CONEXIÓN PROFESIONAL

6. Conexión en Studio 3T

  • Abrir Studio 3T.
  • New Connection.
  • Seleccionar: Paste your connection string.
  • Pegar la URI.
  • Test Connection → Connect.
📸 Evidencia: conexión exitosa mostrando bases disponibles.

7. Error real documentado (si ocurre)

Si aparece:

MongoSecurityException
authentication failed
bad auth

Significa que:

  • Usuario o contraseña incorrectos.
  • URI mal copiada.
  • Problema de permisos.

Solución aplicada:

  • Restablecer contraseña en Database Access.
  • Revisar URI.
  • Probar nuevamente conexión.
📸 Evidencia del error y solución.

PARTE III — CREACIÓN DE BASE PROPIA

8. Crear nueva base de datos

En IntelliShell:

use escuela_4A
⚠ Nota importante: la base no existe hasta que se inserta el primer documento.

PARTE IV — CONSTRUCCIÓN PROGRESIVA (OBLIGATORIO)

⚠ En esta práctica se deberá usar únicamente insertOne(). No usar insertMany().

9. Crear colección

db.createCollection("alumnos")

Verificar:

show collections

10. Insertar documento 1

db.alumnos.insertOne({
  nombre: "Ana López",
  edad: 17,
  grupo: "4-A",
  especialidad: "Programación"
})

Consultar:

db.alumnos.find().pretty()

Observar: campo _id y estructura JSON.

11. Insertar documento 2 (estructura diferente)

db.alumnos.insertOne({
  nombre: "Carlos Ruiz",
  edad: 18,
  grupo: "4-A",
  especialidad: "Programación",
  becado: true
})

Consultar nuevamente y reflexionar: ¿por qué no marca error si los campos no son iguales?

12. Insertar documento 3 (arreglo incluido)

db.alumnos.insertOne({
  nombre: "María Torres",
  edad: 17,
  grupo: "4-A",
  promedio: 9.5,
  materias: ["BDNR", "Programación Web"]
})

Observar: uso de arreglo y flexibilidad estructural.

PARTE V — CONSULTAS BÁSICAS

13. Mostrar todos los documentos

db.alumnos.find().pretty()

14. Filtro por promedio

db.alumnos.find({ promedio: { $gte: 9 } })

Explicar qué hace $gte.
