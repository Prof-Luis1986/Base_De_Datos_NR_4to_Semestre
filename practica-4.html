🟦 Práctica 4

Actualización de documentos JSON usando comandos (updateOne y $set)

🧠 Guía de trabajo

En esta práctica aprenderás a modificar documentos JSON existentes dentro de una colección NoSQL utilizando comandos de actualización. Actualizar datos es una operación delicada: un mal comando puede alterar información importante. Por ello, es fundamental entender qué cambia, qué se conserva y por qué.

Trabajarás sobre la colección usuarios creada en prácticas anteriores usando MongoDB mediante Studio 3T.

🧭 Contexto conceptual

Actualizar datos es más común que crear desde cero. Esta práctica te enseña a modificar campos específicos sin destruir la integridad del documento.

🎯 Aprendizajes esperados

Al finalizar la práctica, serás capaz de:

  • Comprender la diferencia entre insertar y actualizar datos.
  • Usar correctamente updateOne() y el operador $set.
  • Modificar campos específicos sin afectar el resto del documento.
  • Verificar cambios mediante consultas.
  • Explicar los riesgos de una actualización mal aplicada.

🪑 Actividades en salón

(Análisis conceptual – sin computadora)

Analiza este documento JSON:

{
  "nombre": "María",
  "apellido": "García",
  "edad": 25
}

Responde:

  • ¿Qué pasaría si María cambia de edad?
  • ¿Se debe borrar el documento y volverlo a crear?
  • ¿Qué campos NO deberían cambiar?

Reflexiona: actualizar datos no significa reemplazar todo el documento.

📌 Producto del salón: Respuesta escrita razonada (mínimo 8 líneas).

💻 Actividades en laboratorio

(MongoDB – comandos de actualización)

Parte A – Consulta previa

db.usuarios.find().pretty();

Elige un documento a modificar (identifica su _id).

Parte B – Actualización con $set

db.usuarios.updateOne(
  { nombre: "María" },
  { $set: { edad: 26 } }
);

 db.usuarios.find({ nombre: "María" }).pretty();

Parte C – Actualización por _id

db.usuarios.updateOne(
  { _id: ObjectId("PEGA_AQUI_EL_ID_REAL") },
  { $set: { apellido: "Gómez" } }
);

🧪 Practicario

Parte A – Comprensión de comandos

  • ¿Qué hace updateOne()?
  • ¿Para qué sirve $set?

Parte B – Actualizaciones obligatorias

  • Actualiza la edad de dos usuarios distintos.
  • Actualiza el apellido de un usuario.
  • Verifica cada cambio con find().

Parte C – Análisis crítico

Responde:

  • ¿Qué pasaría si no usaras $set?
  • ¿Por qué es más seguro actualizar por _id?

🧩 Justificación técnica de los ejercicios

Los ejercicios con updateOne y $set obligan a entender actualizaciones parciales y el riesgo de reemplazar documentos completos.

🌍 Escenarios reales de aplicación

  • E-commerce: catálogo, carrito y historial de compras.
  • Streaming: perfiles, reproducciones y recomendaciones.
  • Redes sociales: publicaciones, comentarios y reacciones.
  • Sistema escolar: alumnos, materias y evaluaciones.
  • Videojuegos: progreso, logros y sesiones de juego.

🔁 Comparación breve con modelo relacional

En SQL se usan UPDATE con SET; en MongoDB se usan operadores que modifican solo partes del documento.

⚙️ Rendimiento y escalabilidad

  • Evita escanear colecciones completas cuando un índice puede ayudar.
  • Limita y proyecta campos para reducir transferencia de datos.
  • Cuida el tamaño del documento y el crecimiento de arreglos.
  • Mide consultas críticas y ajusta filtros antes de escalar.
  • Usa paginación o límites en lecturas de alto volumen.

🧪 Qué ocurre internamente en MongoDB

MongoDB aplica los modificadores, reescribe el documento si es necesario y actualiza índices asociados a los campos cambiados.

❗ Errores Comunes

  • Actualizar o eliminar sin filtro específico (o sin _id).
  • Guardar tipos incorrectos (números como texto, fechas sin formato).
  • Diseñar documentos demasiado grandes o con campos duplicados.
  • No documentar la estructura JSON y sus cambios.
  • Confundir colección con documento o arreglo con objeto.

✅ Buenas Prácticas Profesionales

  • Definir convenciones de nombres y mantenerlas en todo el proyecto.
  • Validar datos antes de guardar (tipo, rango, formato).
  • Usar índices en campos consultados con frecuencia.
  • Probar consultas con datos reales y revisar planes de ejecución.
  • Documentar decisiones de modelado y razones de embebido vs referencia.

🧠 Análisis Técnico y Reflexión

  • ¿Qué diferencia hay entre $set y reemplazar todo el documento?
  • ¿Qué riesgos existen si no usas filtros precisos?
  • ¿Cómo afecta la actualización a los índices?
  • ¿Qué datos nunca deberían actualizarse?

🧭 Retos Aplicados

  • Actualiza múltiples campos en una sola operación y justifica el filtro.
  • Registra el antes y después con evidencia clara.
  • Usa updateMany con cuidado y explica la diferencia.

🚀 Desafío Nivel Experto

  • Implementa una actualización condicional basada en rango de edades.
  • Propón una estrategia de versionado de documentos.
