🟦 Práctica 4

Actualización de documentos JSON usando comandos (updateOne y $set)

🧠 Guía de trabajo

En esta práctica aprenderás a modificar documentos JSON existentes dentro de una colección NoSQL utilizando comandos de actualización. Actualizar datos es una operación delicada: un mal comando puede alterar información importante. Por ello, es fundamental entender qué cambia, qué se conserva y por qué.

Trabajarás sobre la colección usuarios creada en prácticas anteriores usando MongoDB mediante Studio 3T.

🎯 Aprendizajes esperados

Al finalizar la práctica, serás capaz de:

  • Comprender la diferencia entre insertar y actualizar datos.
  • Usar correctamente updateOne() y el operador $set.
  • Modificar campos específicos sin afectar el resto del documento.
  • Verificar cambios mediante consultas.
  • Explicar los riesgos de una actualización mal aplicada.

🪑 Actividades en salón

(Análisis conceptual – sin computadora)

Analiza este documento JSON:

{
  "nombre": "María",
  "apellido": "García",
  "edad": 25
}

Responde:

  • ¿Qué pasaría si María cambia de edad?
  • ¿Se debe borrar el documento y volverlo a crear?
  • ¿Qué campos NO deberían cambiar?

Reflexiona: actualizar datos no significa reemplazar todo el documento.

📌 Producto del salón: Respuesta escrita razonada (mínimo 8 líneas).

💻 Actividades en laboratorio

(MongoDB – comandos de actualización)

Parte A – Consulta previa

db.usuarios.find().pretty();

Elige un documento a modificar (identifica su _id).

Parte B – Actualización con $set

db.usuarios.updateOne(
  { nombre: "María" },
  { $set: { edad: 26 } }
);

 db.usuarios.find({ nombre: "María" }).pretty();

Parte C – Actualización por _id

db.usuarios.updateOne(
  { _id: ObjectId("PEGA_AQUI_EL_ID_REAL") },
  { $set: { apellido: "Gómez" } }
);

🧪 Practicario

Parte A – Comprensión de comandos

  • ¿Qué hace updateOne()?
  • ¿Para qué sirve $set?

Parte B – Actualizaciones obligatorias

  • Actualiza la edad de dos usuarios distintos.
  • Actualiza el apellido de un usuario.
  • Verifica cada cambio con find().

Parte C – Análisis crítico

Responde:

  • ¿Qué pasaría si no usaras $set?
  • ¿Por qué es más seguro actualizar por _id?
