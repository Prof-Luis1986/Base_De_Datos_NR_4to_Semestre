🟦 Práctica 6

Firebase: Firestore, autenticación con Google y reglas de seguridad

En esta práctica configurarás un proyecto en Firebase, crearás una base de datos en Firestore, habilitarás el inicio de sesión con Google, validarás el acceso desde una página web en HTML y JavaScript, y analizarás cómo las reglas de seguridad controlan el acceso a la información.

🎯 Propósito

Firebase separa claramente tres componentes: Authentication para identidad, Firestore para datos y Security Rules para autorización. La meta es comprender cómo se conectan sin confundir sus funciones.

✅ Aprendizaje esperado

  • Crea un proyecto en Firebase.
  • Habilita Firestore e identifica colección, documento y campo.
  • Activa el acceso con Google en Authentication.
  • Registra una app web y prueba el inicio de sesión.
  • Reconoce el UID como identificador único del usuario.
  • Analiza y modifica reglas de seguridad en Firestore.
  • Comprende por qué la seguridad no depende solo del login.

⏱️ Duración sugerida

2 a 3 sesiones.

🧰 Materiales

  • Cuenta de Google.
  • Navegador web.
  • Editor de código.
  • Conexión a internet.
  • Proyecto de Firebase.

🧠 Introducción

Firebase es una plataforma que permite integrar en una misma solución varios servicios, entre ellos base de datos, autenticación y seguridad. En esta práctica trabajarás con Cloud Firestore, Firebase Authentication y Security Rules. Firestore es una base de datos NoSQL orientada a documentos; Authentication identifica usuarios; y las reglas determinan qué puede hacer cada usuario sobre la información.

📁 Fase 1. Creación del proyecto y de la base de datos

Actividad 1. Crear el proyecto

  • Entra a la consola de Firebase.
  • Selecciona la opción para crear un proyecto nuevo.
  • Asigna un nombre al proyecto.
  • Finaliza el asistente.
Firebase indica que primero debes crear el proyecto y después registrar la app que se conectará a sus servicios.

Actividad 2. Habilitar Firestore

  • Dentro del proyecto, abre el menú Build.
  • Selecciona Firestore Database.
  • Da clic en Create database.
  • Elige el modo inicial indicado por el docente.
  • Selecciona la región.

Actividad 3. Crear una estructura inicial

Crea la colección usuarios y agrega un documento de prueba con estos campos:

  • nombre
  • correo
  • rol

Análisis guiado

Observa la siguiente estructura:

usuarios
  └── alumno01
        ├── nombre: "Luis"
        ├── correo: "luis@email.com"
        └── rol: "estudiante"

Responde en tu cuaderno o documento:

  • ¿Qué parte es la colección?
  • ¿Qué parte es el documento?
  • ¿Qué partes son los campos?
Idea clave: Firestore trabaja con colecciones y documentos, no con tablas y registros como una base de datos relacional.

🔐 Fase 2. Autenticación con Google

Actividad 4. Habilitar Authentication

  • Entra al menú Build.
  • Selecciona Authentication.
  • Da clic en Get started.

Actividad 5. Activar el proveedor Google

  • En la pestaña Sign-in method, localiza Google.
  • Ábrelo.
  • Activa el proveedor.
  • Selecciona el correo de soporte.
  • Guarda los cambios.
Firebase documenta que para usar Google Sign-In en una app web, primero debe habilitarse el proveedor en Authentication.

Actividad 6. Registrar la app web

  • Ve a la vista principal del proyecto.
  • Pulsa el ícono de Web </>.
  • Escribe un nombre para la app, por ejemplo: prueba-auth-web.
  • Registra la app.
  • Copia el objeto de configuración que Firebase te entrega.

Ejemplo de configuración:

const firebaseConfig = {
  apiKey: "TU_API_KEY",
  authDomain: "TU_PROYECTO.firebaseapp.com",
  projectId: "TU_PROYECTO",
  storageBucket: "TU_PROYECTO.firebasestorage.app",
  messagingSenderId: "XXXXXXXX",
  appId: "XXXXXXXX"
};
Firebase explica que al registrar una app web te proporciona la configuración necesaria para inicializar el SDK en JavaScript.

💻 Fase 3. Sistema de prueba en HTML y JavaScript

Actividad 7. Crear el sistema de prueba

Crea un archivo llamado index.html y pega el siguiente código. Después reemplaza los datos de configuración con los de tu proyecto. 

<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>Prueba Firebase Auth con Google</title>
  <style>
    body {
      font-family: Arial, sans-serif;
      max-width: 900px;
      margin: 40px auto;
      padding: 20px;
      line-height: 1.6;
    }
    button {
      margin-right: 10px;
      padding: 10px 16px;
      cursor: pointer;
    }
    .panel {
      margin-top: 20px;
      padding: 16px;
      border: 1px solid #ccc;
      border-radius: 10px;
      background: #f7f7f7;
      white-space: pre-wrap;
    }
  </style>
</head>
<body>
  <h1>Prueba de autenticación con Google y Firebase</h1>
  <p>Esta página permite iniciar sesión con Google y mostrar los datos básicos del usuario autenticado.</p>

  <button id="btnLogin">Iniciar sesión con Google</button>
  <button id="btnLogout">Cerrar sesión</button>

  <div class="panel" id="salida">Aún no hay sesión iniciada.</div>

  <script type="module">
    import { initializeApp } from "https://www.gstatic.com/firebasejs/11.0.0/firebase-app.js";
    import {
      getAuth,
      GoogleAuthProvider,
      signInWithPopup,
      signOut,
      onAuthStateChanged
    } from "https://www.gstatic.com/firebasejs/11.0.0/firebase-auth.js";

    const firebaseConfig = {
      apiKey: "TU_API_KEY",
      authDomain: "TU_PROYECTO.firebaseapp.com",
      projectId: "TU_PROYECTO",
      storageBucket: "TU_PROYECTO.firebasestorage.app",
      messagingSenderId: "TU_MESSAGING_SENDER_ID",
      appId: "TU_APP_ID"
    };

    const app = initializeApp(firebaseConfig);
    const auth = getAuth(app);
    const provider = new GoogleAuthProvider();

    const btnLogin = document.getElementById("btnLogin");
    const btnLogout = document.getElementById("btnLogout");
    const salida = document.getElementById("salida");

    btnLogin.addEventListener("click", async () => {
      try {
        const result = await signInWithPopup(auth, provider);
        const user = result.user;

        salida.textContent =
\`Sesión iniciada correctamente

Nombre: \${user.displayName}
Correo: \${user.email}
UID: \${user.uid}
Proveedor: \${user.providerData[0]?.providerId}\`;
      } catch (error) {
        salida.textContent =
\`Error al iniciar sesión:
\${error.message}\`;
      }
    });

    btnLogout.addEventListener("click", async () => {
      await signOut(auth);
      salida.textContent = "Sesión cerrada.";
    });

    onAuthStateChanged(auth, (user) => {
      if (user) {
        salida.textContent =
\`Sesión activa

Nombre: \${user.displayName}
Correo: \${user.email}
UID: \${user.uid}\`;
      } else {
        salida.textContent = "Aún no hay sesión iniciada.";
      }
    });
  </script>
</body>
</html>
Firebase recomienda en web usar el SDK de JavaScript con GoogleAuthProvider, signInWithPopup() y una inicialización previa del SDK.

Actividad 8. Probar el sistema

  • Abre el archivo en el navegador.
  • Da clic en Iniciar sesión con Google.
  • Selecciona tu cuenta.
  • Observa en pantalla: nombre, correo y UID.
  • Cierra sesión con el botón correspondiente.

Actividad 9. Verificar en Firebase

  • Regresa a Firebase.
  • Abre Authentication.
  • Entra a la sección Users.
  • Comprueba que el usuario aparezca registrado.

Reflexión guiada

  • ¿Dónde apareció el usuario después del login?
  • ¿Apareció automáticamente en Firestore?
  • ¿Qué diferencia notas entre Authentication y Firestore?
Idea clave: Authentication registra la identidad del usuario, pero eso no significa que Firestore cree automáticamente un documento con sus datos.

🛡️ Fase 4. Reglas de seguridad en Firestore

Actividad 10. Localizar las reglas

  • Abre Firestore Database.
  • Entra a la pestaña Rules.

Actividad 11. Analizar una regla abierta

Observa esta regla:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}

La sintaxis de reglas utiliza rules_version = '2';, una declaración service cloud.firestore y condiciones booleanas que determinan si una operación se permite o se niega.

Actividad 12. Bloquear todo

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if false;
    }
  }
}

Actividad 13. Permitir solo usuarios autenticados

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if request.auth != null;
    }
  }
}

En las reglas, request.auth contiene información del usuario autenticado y es null cuando no hay sesión.

Actividad 14. Restringir acceso por UID

Usa esta regla para la colección usuarios:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /usuarios/{userId} {
      allow read, write: if request.auth != null
                         && request.auth.uid == userId;
    }
  }
}

Esta condición compara el UID del usuario autenticado con el identificador del documento. Si no coinciden, el acceso se niega.

Authentication identifica al usuario, el UID lo representa de forma única y las Rules deciden si puede acceder.
