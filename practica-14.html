🟦 Práctica 14

Consultas sobre Arreglos y Documentos Anidados

Modalidad: Laboratorio · Duración: 2 sesiones

🎯 Propósito

Aprender a consultar información dentro de arreglos y objetos anidados.

🧭 Contexto conceptual

Consultar dentro de arreglos y documentos anidados es una habilidad clave para explotar el potencial documental de MongoDB.

🧠 Competencias

  • Consulta datos dentro de arreglos.
  • Filtra por propiedades internas.
  • Utiliza operadores especiales para arreglos.

📚 Parte 1 — Consultar dentro de arreglos

Mostrar usuarios que tengan en su historial la película “Matrix”:

db.usuarios.find({
  "historial.titulo": "Matrix"
})

📚 Parte 2 — Operador $elemMatch

Buscar usuarios cuyo historial tenga película con minutos mayores a 100:

db.usuarios.find({
  historial: {
    $elemMatch: { minutos_vistos: { $gt: 100 } }
  }
})

📚 Parte 3 — Consultas en documentos anidados

Buscar usuarios que vivan en CDMX:

db.usuarios.find({
  "direccion.ciudad": "CDMX"
})

🧠 Parte 4 — Análisis

Responder:

  • ¿Qué diferencia hay entre buscar directamente y usar $elemMatch?
  • ¿Por qué las consultas anidadas son importantes?
  • ¿En qué tipo de aplicaciones esto es indispensable?

📦 Entregables

  • Capturas.
  • Código.
  • Reflexión técnica.

🧩 Justificación técnica de los ejercicios

Los ejercicios con $elemMatch y campos anidados enseñan a filtrar datos en estructuras reales con precisión.

🌍 Escenarios reales de aplicación

  • E-commerce: catálogo, carrito y historial de compras.
  • Streaming: perfiles, reproducciones y recomendaciones.
  • Redes sociales: publicaciones, comentarios y reacciones.
  • Sistema escolar: alumnos, materias y evaluaciones.
  • Videojuegos: progreso, logros y sesiones de juego.

🔁 Comparación breve con modelo relacional

En SQL necesitarías JOINs para consultar estructuras relacionadas; en MongoDB puedes filtrar dentro del mismo documento.

⚙️ Rendimiento y escalabilidad

  • Evita escanear colecciones completas cuando un índice puede ayudar.
  • Limita y proyecta campos para reducir transferencia de datos.
  • Cuida el tamaño del documento y el crecimiento de arreglos.
  • Mide consultas críticas y ajusta filtros antes de escalar.
  • Usa paginación o límites en lecturas de alto volumen.

🧪 Qué ocurre internamente en MongoDB

MongoDB recorre arreglos y evalúa condiciones; $elemMatch asegura que se cumplan múltiples condiciones en el mismo elemento.

❗ Errores Comunes

  • Actualizar o eliminar sin filtro específico (o sin _id).
  • Guardar tipos incorrectos (números como texto, fechas sin formato).
  • Diseñar documentos demasiado grandes o con campos duplicados.
  • No documentar la estructura JSON y sus cambios.
  • Confundir colección con documento o arreglo con objeto.

✅ Buenas Prácticas Profesionales

  • Definir convenciones de nombres y mantenerlas en todo el proyecto.
  • Validar datos antes de guardar (tipo, rango, formato).
  • Usar índices en campos consultados con frecuencia.
  • Probar consultas con datos reales y revisar planes de ejecución.
  • Documentar decisiones de modelado y razones de embebido vs referencia.

🧠 Análisis Técnico y Reflexión

  • ¿Qué problema resuelve $elemMatch?
  • ¿Qué diferencia hay entre filtros en objeto vs arreglo?
  • ¿Cómo afecta la consulta a rendimiento en arreglos grandes?
  • ¿Qué casos reales requieren este tipo de consultas?

🧭 Retos Aplicados

  • Crea una consulta con múltiples condiciones dentro de historial.
  • Compara resultados entre consulta directa y $elemMatch.
  • Busca usuarios por rango de edad dentro de perfiles.

🚀 Desafío Nivel Experto

  • Diseña un filtro de historial con ventana de fechas y justifícalo.
  • Propón índices para consultas anidadas frecuentes.
