Unidad II · Modelado Documental y Estructura de Datos con JSON
🟦 Práctica 8

Lectura y visualización de JSON

Leer documentos y mostrarlos en pantalla con claridad.

📘 Guía de trabajo

El alumno consultará documentos en Firestore y mostrará la información en una interfaz, reforzando el ciclo completo: guardar, leer y visualizar datos.

No basta con guardar datos: el verdadero valor está en poder leerlos y mostrarlos correctamente.

🧭 Contexto conceptual

Leer y visualizar JSON correctamente es la base para depurar errores y validar datos. Esta práctica entrena tu capacidad para interpretar estructuras complejas.

🎯 Objetivos de aprendizaje

  • Leer documentos desde Firestore.
  • Recorrer los campos y mostrarlos en HTML.
  • Separar claramente la capa de datos y la capa de presentación.

🧪 Practicario (entregable)

  • Leer al menos un documento o una colección pequeña.
  • Mostrar los datos en una lista, tabla o tarjetas.
  • Incluir un estado de “sin datos” o “cargando” si es posible.

🧩 Buenas prácticas sugeridas

  • Mostrar solo los campos relevantes para el usuario.
  • Formatear números, fechas y textos largos cuando sea necesario.
  • Evitar mezclar demasiada lógica dentro del HTML.

🎨 Producto digital

Infografía de flujo: captura → JSON → Firestore → lectura → visualización.

✅ Evidencias para Google Sites

  • Captura del resultado visual.
  • Captura o fragmento del código de lectura.
  • Infografía final.

🧩 Justificación técnica de los ejercicios

Los ejercicios de lectura te obligan a identificar tipos de datos, niveles de anidación y coherencia en la estructura.

🌍 Escenarios reales de aplicación

  • E-commerce: catálogo, carrito y historial de compras.
  • Streaming: perfiles, reproducciones y recomendaciones.
  • Redes sociales: publicaciones, comentarios y reacciones.
  • Sistema escolar: alumnos, materias y evaluaciones.
  • Videojuegos: progreso, logros y sesiones de juego.

🔁 Comparación breve con modelo relacional

En SQL la lectura se hace en filas y columnas; en MongoDB debes interpretar objetos y arreglos con diferentes niveles.

⚙️ Rendimiento y escalabilidad

  • Evita escanear colecciones completas cuando un índice puede ayudar.
  • Limita y proyecta campos para reducir transferencia de datos.
  • Cuida el tamaño del documento y el crecimiento de arreglos.
  • Mide consultas críticas y ajusta filtros antes de escalar.
  • Usa paginación o límites en lecturas de alto volumen.

🧪 Qué ocurre internamente en MongoDB

MongoDB devuelve documentos BSON convertidos a JSON en el cliente, conservando tipos como ObjectId y Date para su correcta interpretación.

❗ Errores Comunes

  • Actualizar o eliminar sin filtro específico (o sin _id).
  • Guardar tipos incorrectos (números como texto, fechas sin formato).
  • Diseñar documentos demasiado grandes o con campos duplicados.
  • No documentar la estructura JSON y sus cambios.
  • Confundir colección con documento o arreglo con objeto.

✅ Buenas Prácticas Profesionales

  • Definir convenciones de nombres y mantenerlas en todo el proyecto.
  • Validar datos antes de guardar (tipo, rango, formato).
  • Usar índices en campos consultados con frecuencia.
  • Probar consultas con datos reales y revisar planes de ejecución.
  • Documentar decisiones de modelado y razones de embebido vs referencia.

🧠 Análisis Técnico y Reflexión

  • ¿Qué problemas aparecen si un JSON no tiene formato consistente?
  • ¿Cómo detectarías valores nulos peligrosos?
  • ¿Qué campos son candidatos a indexarse?
  • ¿Qué decisiones de lectura impactan el rendimiento?

🧭 Retos Aplicados

  • Identifica inconsistencias en un JSON grande y propón correcciones.
  • Distingue campos opcionales vs obligatorios en una muestra real.
  • Propón un esquema mínimo para validar ese JSON.

🚀 Desafío Nivel Experto

  • Diseña una visualización que muestre jerarquías de un JSON complejo.
  • Explica cómo mapearías ese JSON a un modelo relacional.
