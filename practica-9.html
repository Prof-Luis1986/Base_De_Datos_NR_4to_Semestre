🟦 Práctica 9

Tutorial del alumno: registro de plantas con Flet

Modalidad: laboratorio guiado · Propósito: adaptar una interfaz en Flet para capturar datos reales y conectarla con MongoDB o Firebase sin modificar el diseño visual base.

Esta práctica no pide rediseñar la app. El trabajo del alumno consiste en leer el formulario, validar los datos y guardar registros reales en una base de datos.

1. Objetivo de la práctica

La aplicación ya tiene la parte visual construida. El alumno debe enfocarse en la lógica de persistencia.

  • Ya existe: panel lateral, encabezado, formulario y tabla de ejemplo.
  • Debe desarrollar: lectura de campos, validación, conexión y guardado real.
  • Resultado esperado: mostrar en pantalla registros obtenidos desde MongoDB o Firebase.

3. Actividad de investigación en el patio

Antes de capturar información en la app, el alumno debe observar plantas reales del patio, jardín, huerto escolar o zona permitida por el docente.

  • Propósito: relacionar programación con observación del entorno real.
  • Materiales: teléfono móvil o cuaderno, lápiz o pluma y cámara si se permite.
  • Regla clave: no inventar datos; registrar solo lo observado.

El propósito de esta actividad es relacionar la programación con una investigación real del entorno, para que los registros no sean inventados sino obtenidos mediante observación.

Materiales:

  • Teléfono móvil o cuaderno.
  • Lápiz o pluma.
  • Cámara del teléfono si el docente permite tomar evidencia.

3.1 Instrucciones de trabajo de campo

  • Salir al patio o zona asignada por el docente.
  • Observar con atención las plantas que se encuentren en ese espacio.
  • Elegir varias plantas diferentes para registrarlas después en la aplicación.
  • Anotar cada planta en el teléfono o en el cuaderno.
  • Registrar la información más completa posible sin inventar datos.

3.2 Datos mínimos que debe anotar el alumno

Datos mínimos para anotar:

  • Nombre común de la planta, si se conoce.
  • Características visibles como tamaño, color o tipo de hoja.
  • Ubicación donde fue encontrada.
  • Estado observable, por ejemplo sana, seca o floreciendo.
  • Fecha de observación.

3.3 Si el alumno no conoce el nombre de la planta

Si no reconoce la especie, debe anotar una descripción provisional. Por ejemplo: "planta de hojas alargadas cerca del jardín central". Después puede investigar el nombre común o científico con apoyo del docente o de una fuente confiable.

4. Cómo ejecutar la app

La ejecución sugerida es en modo web para conservar el comportamiento esperado de Flet.

source .venv/bin/activate
flet run --web src/main.py
Si se ejecuta con python src/main.py, Flet puede abrirse en otro modo y no coincidir con la experiencia planeada para la práctica.

5. Lectura guiada del código actual

El archivo src/main.py debe leerse por bloques para identificar qué parte es visual y qué parte debe completarse.

  • import flet as ft: importa la librería principal de interfaz.
  • stat_card(): crea tarjetas visuales reutilizables.
  • label_value(): genera pares de etiqueta y valor en columna.
  • main(page): arma la página completa y define tema, scroll y layout.
  • mostrar_aviso(): hoy solo muestra mensajes temporales; debe reemplazarse por lógica real.
  • form_section: contiene los controles que el alumno debe convertir en variables.
  • table_section: muestra filas simuladas que luego deberán provenir de la base de datos.
  • ft.run(main): inicia la aplicación.

7. Conexión con MongoDB

MongoDB es una base documental NoSQL y en Python suele usarse mediante pymongo.

pip install pymongo
  • Tener cuenta en MongoDB Atlas o servidor local.
  • Crear base de datos y colección, por ejemplo plantas.
  • Usar una cadena de conexión válida.
  • Verificar permisos de lectura y escritura.
from pymongo import MongoClient

MONGO_URI = "mongodb+srv://USUARIO:CLAVE@cluster0.xxxxx.mongodb.net/?retryWrites=true&w=majority"

client = MongoClient(MONGO_URI)
db = client["registro_plantas"]
coleccion = db["plantas"]

def guardar_en_mongo(datos):
    resultado = coleccion.insert_one(datos)
    return str(resultado.inserted_id)

def obtener_registros_mongo():
    registros = []
    for doc in coleccion.find():
        doc["_id"] = str(doc["_id"])
        registros.append(doc)
    return registros

8. Conexión con Firebase

Para esta práctica se propone trabajar con Cloud Firestore desde Python usando firebase-admin.

pip install firebase-admin
  • Crear un proyecto en Firebase.
  • Habilitar Firestore en modo nativo.
  • Descargar la cuenta de servicio en JSON.
  • Comprobar permisos y nombre de la colección.
import firebase_admin
from firebase_admin import credentials, firestore

cred = credentials.Certificate("credenciales/firebase-key.json")
firebase_admin.initialize_app(cred)
db = firestore.client()

def guardar_en_firebase(datos):
    ref = db.collection("plantas").add(datos)
    return str(ref[1].id)

def obtener_registros_firebase():
    registros = []
    docs = db.collection("plantas").stream()
    for doc in docs:
        item = doc.to_dict()
        item["id"] = doc.id
        registros.append(item)
    return registros

9. Cómo decidir entre MongoDB y Firebase

Opción Conviene cuando Punto de atención
MongoDB Se quiere trabajar con documentos JSON y consultas flexibles. Debe cuidarse la cadena de conexión y los permisos.
Firebase Se busca una solución administrada con servicios de Google. Debe protegerse el archivo JSON de credenciales.

10. Lógica general para guardar

def guardar_registro(e):
    error = validar_formulario()
    if error:
        page.snack_bar = ft.SnackBar(content=ft.Text(error))
        page.snack_bar.open = True
        page.update()
        return

    datos = obtener_datos_formulario()

    try:
        if base_datos.value == "MongoDB":
            guardar_en_mongo(datos)
        elif base_datos.value == "Firebase":
            guardar_en_firebase(datos)
        else:
            raise ValueError("Selecciona una base de datos.")

        page.snack_bar = ft.SnackBar(content=ft.Text("Registro guardado correctamente."))
        page.snack_bar.open = True
        page.update()
    except Exception as ex:
        page.snack_bar = ft.SnackBar(content=ft.Text(f"Error al guardar: {ex}"))
        page.snack_bar.open = True
        page.update()

Las filas fijas deben reemplazarse por una lista construida dinámicamente con los documentos guardados.

def construir_filas(registros):
    filas = []
    for item in registros:
        filas.append(
            ft.DataRow(
                cells=[
                    ft.DataCell(ft.Text(item.get("nombre_comun", ""))),
                    ft.DataCell(ft.Text(item.get("tipo_planta", ""))),
                    ft.DataCell(ft.Text(item.get("alumno_responsable", ""))),

                ]
            )
        )
    return filas

13. Errores comunes

  • No guardar referencias a los campos del formulario.
  • Intentar leer .value de controles no asignados a variables.
  • Escribir credenciales dentro del código y compartirlas.
  • No capturar excepciones de conexión.
  • Olvidar page.update() después de modificar la interfaz.
  • Confundir Firestore con Realtime Database.
  • No convertir ObjectId a texto al mostrar datos de MongoDB.
