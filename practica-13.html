Unidad III · Operaciones y Manipulación de Datos en MongoDB
🟦 Práctica 13

Actualizaciones Avanzadas y Manipulación de Arreglos

Modalidad: Laboratorio · Duración: 2 sesiones

🎯 Propósito

Dominar operadores avanzados de actualización y manipular arreglos dentro de documentos en MongoDB.

🧭 Contexto conceptual

Actualizar arreglos es una necesidad diaria en aplicaciones reales: listas de teléfonos, historial, preferencias. Esta práctica profundiza en esas operaciones.

🧠 Competencias

  • Utiliza operadores $push, $pull, $inc, $set.
  • Manipula arreglos correctamente.
  • Comprende cómo modificar estructuras anidadas.

🔎 Contexto

En aplicaciones reales, los documentos cambian constantemente. No basta con modificar un campo simple: hay que agregar elementos a arreglos, eliminar objetos dentro de arreglos y actualizar valores internos.

📚 Parte 1 — Operadores Básicos de Actualización

Trabajar con colección usuarios.

1️⃣ Agregar campo nuevo

db.usuarios.updateOne(
  { nombre: "Carlos" },
  { $set: { membresia: "Gold" } }
)

2️⃣ Incrementar valor

db.usuarios.updateOne(
  { nombre: "Carlos" },
  { $inc: { edad: 1 } }
)

📚 Parte 2 — Manipulación de Arreglos

3️⃣ Agregar elemento a arreglo

db.usuarios.updateOne(
  { nombre: "Carlos" },
  { $push: { telefonos: "5511111111" } }
)

4️⃣ Eliminar elemento del arreglo

db.usuarios.updateOne(
  { nombre: "Carlos" },
  { $pull: { telefonos: "5511111111" } }
)

📚 Parte 3 — Actualizar objetos dentro de arreglos

Modificar un elemento del historial:

db.usuarios.updateOne(
  { nombre: "Carlos", "historial.titulo": "Matrix" },
  { $set: { "historial.$.minutos_vistos": 120 } }
)

🧠 Parte 4 — Análisis

Responder:

  • ¿Qué ventaja tiene usar $push en lugar de reescribir el arreglo completo?
  • ¿Qué riesgo existe al usar mal $pull?
  • ¿Por qué MongoDB permite modificar partes internas del documento?

  • Capturas.
  • Reflexión técnica.

🧩 Justificación técnica de los ejercicios

Los ejercicios con $push y $pull muestran cómo modificar arreglos sin reescribir documentos completos.

🌍 Escenarios reales de aplicación

  • E-commerce: catálogo, carrito y historial de compras.
  • Streaming: perfiles, reproducciones y recomendaciones.
  • Redes sociales: publicaciones, comentarios y reacciones.
  • Sistema escolar: alumnos, materias y evaluaciones.
  • Videojuegos: progreso, logros y sesiones de juego.

🔁 Comparación breve con modelo relacional

En SQL, arreglos suelen normalizarse en tablas; en MongoDB se pueden embebir y actualizar con operadores.

⚙️ Rendimiento y escalabilidad

  • Evita escanear colecciones completas cuando un índice puede ayudar.
  • Limita y proyecta campos para reducir transferencia de datos.
  • Cuida el tamaño del documento y el crecimiento de arreglos.
  • Mide consultas críticas y ajusta filtros antes de escalar.
  • Usa paginación o límites en lecturas de alto volumen.

🧪 Qué ocurre internamente en MongoDB

MongoDB aplica cambios sobre el documento completo y actualiza índices de campos modificados; en arreglos grandes, el costo puede crecer.

❗ Errores Comunes

  • Actualizar o eliminar sin filtro específico (o sin _id).
  • Guardar tipos incorrectos (números como texto, fechas sin formato).
  • Diseñar documentos demasiado grandes o con campos duplicados.
  • No documentar la estructura JSON y sus cambios.
  • Confundir colección con documento o arreglo con objeto.

✅ Buenas Prácticas Profesionales

  • Definir convenciones de nombres y mantenerlas en todo el proyecto.
  • Validar datos antes de guardar (tipo, rango, formato).
  • Usar índices en campos consultados con frecuencia.
  • Probar consultas con datos reales y revisar planes de ejecución.
  • Documentar decisiones de modelado y razones de embebido vs referencia.

🧠 Análisis Técnico y Reflexión

  • ¿Qué ventaja real tiene $push frente a reescribir el arreglo?
  • ¿Qué pasa si un arreglo crece sin control?
  • ¿Cómo afecta la actualización a los índices?
  • ¿Qué estrategia usarías para evitar duplicados?

🧭 Retos Aplicados

  • Agrega un objeto con más campos al historial y actualízalo.
  • Usa $inc y $set en una sola operación con un filtro preciso.
  • Justifica cuándo conviene usar $addToSet en lugar de $push.

  • Explica cómo auditarías cambios en arreglos.
